Inleiding

Tijdens het bouwen van de app, maak ik gebruik van user stories en de MoSCoW methode om duidelijk te krijgen waar ik nog aan moet werken. Door bepaalde elementen van de app in de “Must” van de methode te zetten, weet ik dat ik daar als eerste naar moet kijken.
Voor mijn gedeelte van de app was dat de login pagina en het geven en lezen van een review bij een recept.

Tijdens het werken aan de app, werkte ik met GIT in een aparte ‘branch’, zodat mijn bijdrage en volledige code hier ook te lezen is:
https://github.com/luukvanettinger/simac-frontend/tree/loginstyle
https://github.com/luukvanettinger/simac-frontend/tree/review

Aanpak

Ik werkte tijdens de app aan de volgende user stories:
- Als Simac medewerker wil ik een recept een review geven met opmerking, zodat ik duidelijk kan maken aan mijn collega’s wat er wel of niet goed is aan een recept
Acceptatie criteria:
- Review geven aan een recept
- Opmerking bij het reviewen van een plaatsen

- Als Simac werknemer wil ik kunnen zien welk recept goed gewaardeerd wordt, zodat ik weet welk recept lekker is.
Acceptatie criteria
- Reviewen van een recept
Een recept moet aangeklikt kunnen worden en details laten zien

- Als Simac medewerker wil ik bepaalde recepten van een gekozen thema kunnen kiezen, zodat ik recepten kan vinden passen bij mijn gekozen moment.
Acceptatie criteria
- Thema’s die passen bij Simac medewerkers
- Recepten filteren

Resultaat

Ik heb eerst een ‘state’ aangemaakt. Hiermee kun je data opslaan wat later door een gebruiker toegevoegd kan worden.
useEffect-functie wordt aangeroepen wanneer een pagina wordt geladen of een actie wordt uitgevoerd. Dus wanneer de pagina wordt geopend, worden de reviews geopend en getoond.


Daarna heb ik code toegevoegd waarmee ik data uit de Firebase haal en kan laten zien op de pagina. De reviews staan op de Firebase achter de recepten. Dus eerst roep ik de 'collection' aan waar de recepten staan, dan naar de 'id' van een recept, en dan bevat dat specifieke recept weer een collection aan reviews.

Filteren van recepten
Op de receptenpagina heb ik ook nog een pagina gemaakt waar je alle recepten kunt vinden. De recepten kun je hier op thema bekijken. De recepten passen bij bepaalde aspecten die het welzijn verbeteren maar vooral terugkomen bij het concept: betrokkenheid en verbondenheid creëren (Jumbo, 2021).

In een later stadium, wanneer veel mensen gebruik maken van de app, kan Simac kiezen voor meer gezondere recepten, zodat ze zich meer gaan richten op het welzijn.

Code uitleg
In de state heb ik undefined aangegeven, want de filter moet nog niet geactiveerd zijn bij het openen van de pagina. Wanneer je vervolgens op een thema klikt, voert hij die filter in bij de state, en zoekt hij alle recepten met het gekozen thema.

‘let’ is hetzelfde als een ‘const’, alleen kan deze nog aangepast worden omdat ik hier de filter in ga zetten. Op basis van de state wordt de let ingevuld
